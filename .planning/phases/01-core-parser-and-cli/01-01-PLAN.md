<threat_model>
Assets: local Codex session logs and any files near the user's Codex home.
Threats:
- Accidental credential access if discovery scans the whole Codex home.
- Private prompt leakage if tests copy real logs.
- Network exfiltration if usage data is sent to an API.
Mitigations:
- Implement discovery as an allowlist for `sessions` and `archived_sessions`.
- Use only synthetic fixtures, including a dummy `auth.json` decoy with `DO_NOT_READ`.
- Do not add network dependencies or HTTP calls in Phase 1.
Residual risk: Codex may rename session directories later; this should surface as a no-log diagnostic, not unsafe broad scanning.
</threat_model>
