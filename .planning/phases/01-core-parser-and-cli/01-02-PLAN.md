<threat_model>
Assets: local usage counters and parser diagnostics.
Threats:
- Overcounting cumulative `total_token_usage` events and showing inflated usage.
- Failing all aggregation because one JSONL line is malformed or missing optional fields.
- Accidentally parsing message payloads instead of token metadata.
Mitigations:
- Parse only `token_count` events and their token usage objects.
- Track sessions by session id when available, with file path fallback.
- Add regression tests where totals 100 then 150 produce 150, not 250.
- Return warnings for malformed or incomplete lines while preserving valid records.
Residual risk: If Codex changes field names, Phase 1 reports diagnostics instead of guessing.
</threat_model>
